Introdução

JavaScript é uma linguagem de programação orientada a objeto multi-plataforma. É uma linguagem leve e pequena. Dentro de um ambiente host (por exemplo, um navegador web), JavaScript pode ser conectada aos objetos em seus ambientes para fornecer controle programático sobre eles.

JavaScript contém uma biblioteca padrão de objetos, como Arrays, Data, e Matemática, e um conjunto básico de elementos da linguagem, como operadores, estruturas de controle e instruções. O JavaScript básico pode ser estendido para uma variedade de propósitos, complementando-o com objetos, por exemplo:

  • • No lado do client o JavaScript se estende em fornecer objetos para controlar o navegador e o DOM (Document Object Model). Por exemplo do lado do cliente extensões permitem aplicações que colocam elementos em um formulário HTML e respondem a eventos do usuário, como cliques do mouse, entrada de formulário e navegação de página.

  • • Já no lado do servidor a linguagem fornece objetos relevantes para a execução de JavaScript em um servidor. Por exemplo, as extensões do lado do servidor, permitem que um aplicativo se comunice com o banco de dados, assim fornecendo continuidade de informações de uma chamada para outra do aplicativo ou execute manipulações de arquivo em um servidor.

JavaScript e Java

JavaScript e Java são parecidos em alguns aspectos, porém fundamentalmente diferentes em outros. A linguagem JavaScript se assemelha ao Java, porém não possui uma tipagem estática e veificação de tipo forte como é encontrado no Java. O JavaScript segue a maior parte da sintaxe de expressão Java, convenções de nomenclatura e construções básicas de fluxo de controle, razão pela qual foi renomeado de LiveScript para JavaScript.

Em contraste com o tempo de compilação do sistema de classes do Java construído por declarações, o JavaScript oferece a um sistema de tempo de execução baseado em um pequeno número de tipos de dados que representam valores númericos, boleanos e de string. JavaScript tem um modelo de objeto baseado em protótipo em vez do modelo de objeto em classe encontrado mais comumente. O modelo baseado em protótipo fornece herança dinâmica, ou seja, o que é herdado pode variar para objetos individuais. O JavaScript também oferece suporte a funções sem quaisquer requisitos declarativos especiais. Funções podem ser propriedades de objetos, executando como métodos vagamente tipados.

JavaScript é uma linguagem de formato muito livre em compração com o Java. Você não precisa declarar todas as variváveis, classes e métodos. Você não precisa se preocupar se os metódos são públicos, privados ou protegidos e também não precisa implementar interfaces. Variáveis, parâmetros e tipos de retorno de função não são explicitamente digitados.

Variáveis

Você usa variáveis ​​como nomes simbólicos para valores em seu aplicativo. Os nomes das variáveis, chamados identificadores, obedecem a certas regras.

Um identificador JavaScript deve começar com uma letra, sublinhado (_) ou cifrão ($); os caracteres subsequentes também podem ser dígitos (0-9). Como o JavaScript diferencia maiúsculas de minúsculas, as letras incluem os caracteres "A" a "Z" (maiúsculas) e os caracteres "a" a "z" (minúsculas).

Você pode usar letras ISO 8859-1 ou Unicode, como å e ü em identificadores. Você também pode usar as sequências de escape Unicode como caracteres em identificadores. Alguns exemplos de nomes legais são Number_hits, temp99 e _name.

Declarando variáveis

Você pode declarar vaiáveis de três maneiras:

Com a palavra-chave var. Por exemplo,

var x = 42.

A sintaxe pode ser usada para declarar variáveis globais e locais.

Simplesmente atribuindo um valor a ele. Por exemplo,

x = 42.

Isso sempre declara uma variável global. Ele gera um aviso estrito de JavaScript. Você não deve usar esta variante.

Com a palavra-chave let. Por exemplo,

let y = 13.

Esta sintaxe pode ser usada para declarar uma variável local de escopo de bloco. Leia mais sobre o alcance do escopo da variável abaixo.

Alcance das variáveis

Quando você declara uma variável fora de qualquer função, ela é chamada de variável global, porque está disponível para qualquer outro código no documento atual. Quando você declara uma variável dentro de uma função, ela é chamada de variável local, porque está disponível apenas nessa função.

JavaScript antes do ECMAScript 2015 não tem escopo de instrução de bloco; em vez disso, uma variável declarada dentro de um bloco é local para a função (ou escopo global) em que o bloco reside. Por exemplo, o código a seguir registrará 5, porque o escopo de x é a função (ou contexto global) dentro da qual x é declarado, não o bloco, que neste caso é uma instrução if.

if (true) { var x = 5; } console.log(x); // 5

Este comportamento muda, ao usar a declaração let introduzida no ECMAScript 2015.

if (true) { let y = 5; } console.log(y); // ReferenceError: y is not defined

Variáveis globais

Variáveis ​​globais são na verdade propriedades do objeto global. Em páginas da web, o objeto global é a janela, portanto, você pode definir e acessar variáveis ​​globais usando a sintaxe window.variable.

Consequentemente, você pode acessar variáveis ​​globais declaradas em uma janela ou quadro a partir de outra janela ou quadro, especificando o nome da janela ou quadro. Por exemplo, se uma variável chamada phoneNumber for declarada em um documento, você pode se referir a essa variável em um iframe como parent.phoneNumber.

Constantes

Você pode criar uma constante somente leitura, nomeada com a palavra-chave const. A sintaxe de um identificador de constante é a mesma de um identificador de variável: ele deve começar com uma letra, sublinhado ou cifrão e pode conter caracteres alfabéticos, numéricos ou sublinhados.

const PI = 3.14;

Uma constante não pode alterar o valor por meio de atribuição ou ser declarada novamente durante a execução do script. Ele deve ser inicializado com um valor.

As regras de escopo para constantes são iguais às de permitir variáveis ​​de escopo de bloco. Se a palavra-chave const for omitida, o identificador será considerado uma variável.

Você não pode declarar uma constante com o mesmo nome de uma função ou variável no mesmo escopo. Por exemplo:

// THIS WILL CAUSE AN ERROR function f() {}; const f = 5; // THIS WILL CAUSE AN ERROR ALSO function f() { const g = 5; var g; //statements }

No entanto, os atributos do objeto não são protegidos, portanto, a instrução a seguir é executada sem problemas.

const MY_OBJECT = {"key": "value"}; MY_OBJECT.key = "otherValue";

Tipos de dados

O padrão ECMAScript mais recente define sete tipos de dados:

Seis tipos de dados que são primitivos:

  • Boleano. verdadeiro e falso.

  • null. Uma palavra-chave especial denotando um valor nulo. Como o JavaScript diferencia maiúsculas de minúsculas, null não é o mesmo que Null, NULL ou qualquer outra variante.

  • undifined. Uma propriedade de nível superior cujo valor é indefinido.

  • Number. 42 ou 3,14159.

  • String. "Olá"

  • Symbol (novo no ECMAScript 2015). Um tipo de dados cujas instâncias são exclusivas e imutáveis.

O sétimo tipo é Objeto

Embora esses tipos de dados sejam uma quantidade relativamente pequena, eles permitem que você execute funções úteis com seus aplicativos. Objetos e funções são os outros elementos fundamentais da linguagem. Você pode pensar em objetos como contêineres nomeados para valores e funções como procedimentos que seu aplicativo pode executar.

Declarações if... else

Use a instrução if para executar uma instrução se uma condição lógica for verdadeira. Use a cláusula opcional else para executar uma statement se a condição for falsa. Uma declaração if tem a seguinte aparência:

if (condition) { statement_1; } else { statement_2; }

condição pode ser qualquer expressão avaliada como verdadeira ou falsa. Consulte Boolean para uma explicação do que é avaliado como verdadeiro e falso. Se a condição for avaliada como verdadeira, a statement_1 será executada; caso contrário, a statement_2 é executada. statement_1 e statement_2 podem ser qualquer instrução, incluindo outras instruções if aninhadas.

Você também pode compor as statement usando else if para ter várias condições testadas em sequência, da seguinte maneira:

if (condition_1) { statement_1; } else if (condition_2) { statement_2; } else if (condition_n) { statement_n; } else { statement_last; }

No caso de várias condições, apenas a primeira condição lógica avaliada como verdadeira será executada. Para executar várias statements, agrupe-as em uma statement em bloco ({...}). Em geral, é uma boa prática sempre usar statements de bloco, especialmente ao aninhar statements if:

if (condition) { statement_1_runs_if_condition_is_true; statement_2_runs_if_condition_is_true; } else { statement_3_runs_if_condition_is_false; statement_4_runs_if_condition_is_false; }

É aconselhável não usar atribuições simples em uma expressão condicional, porque a atribuição pode ser confundida com igualdade ao examinar o código. Por exemplo, não use o seguinte código:

if (x = y) { /* statements here */ }

Se você precisar usar uma atribuição em uma expressão condicional, uma prática comum é colocar parênteses adicionais em torno da atribuição. Por exemplo:

if ((x = y)) { /* statements here */ }

Declaração while

Uma statement while executa suas statements enquanto uma condição especificada for avaliada como verdadeira. Uma declaração while tem a seguinte aparência:

while (condition) statement

Se a condição se tornar falsa, a statement dentro do loop para de ser executada e o controle passa para a statement seguinte ao loop.

O teste de condição ocorre antes que a instrução no loop seja executada. Se a condição retornar verdadeira, a statement é executada e a condição é testada novamente. Se a condição retornar falsa, a execução é interrompida e o controle é passado para a statement seguinte ao while.

Para executar várias statements, use uma statement em bloco ({...}) para agrupar essas statements.

Exemplo:

O seguinte loop while itera, desde que n seja menor que três:

var n = 0; var x = 0; while (n < 3) { n++; x += n; }

Com cada iteração, o loop incrementa n e adiciona esse valor a x. Portanto, x e n assumem os seguintes valores:

  • Após a primeira passagem: n = 1 e x = 1

  • Após a segunda passagem: n = 2 e x = 3

  • Após a terceira passagem: n = 3 e x = 6

Depois de completar a terceira passagem, a condição n <3 não é mais verdadeira, então o loop termina.

Declaração de função

Uma definição de função (também chamada de declaração de função ou instrução de função) consiste na palavra-chave da função, seguida por:

  • O nome da função.

  • Uma lista de argumentos para a função, colocados entre parênteses e separados por vírgulas.

  • As instruções JavaScript que definem a função, entre chaves, {}.

Por exemplo, o código a seguir define uma função simples chamada square:

function square(number) { return number * number; }

The function square takes one argument, called number. The function consists of one statement that says to return the argument of the function (that is, number) multiplied by itself. The return statement specifies the value returned by the function.

return number * number;

Parâmetros primitivos (como um número) são passados ​​para funções por valor; o valor é passado para a função, mas se a função alterar o valor do parâmetro, essa alteração não será refletida globalmente ou na função de chamada.
